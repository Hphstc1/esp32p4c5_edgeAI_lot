# 技术变更日志

## 2026-07-08：环缓冲区索引计算修复

### 修复内容

在 `wifi_tcp_stream.cpp` 和 `http_server.cpp` 中，`serve_stream()` 和 `serve_snapshot()` 函数用于定位环缓冲区最新帧的索引公式，从 buggy 的 if/else 分支修正为统一公式。

### 环缓冲区数据结构

```cpp
// 固定物理数组，逻辑容量可配 [1, 8]
JpegFrame ring_[8];
int ring_cap_;    // 逻辑容量（默认 3）
int ring_head_;   // 下一次将写入的位置
int ring_fill_;   // 当前有效帧数
```

### push_jpeg 写入逻辑

```cpp
void push_jpeg(const uint8_t *data, size_t len) {
    JpegFrame &slot = ring_[ring_head_];           // [1] 写入当前位置
    slot.data.assign(data, data + len);
    ring_head_ = (ring_head_ + 1) % ring_cap_;     // [2] head 自增 + 回绕
    if (ring_fill_ < ring_cap_) ring_fill_++;       // [3] 记录填充数
    push_seq_.fetch_add(1);                         // [4] 通知等待者
}
```

### 修复前（BUGGY）

```cpp
int idx;
if (ring_fill_ < ring_cap_)
    idx = (ring_head_ - 1 + ring_fill_) % ring_cap_;
else
    idx = (ring_head_ - 1 + ring_cap_) % ring_cap_;
```

当 `ring_fill_ < ring_cap_` 时公式错误：用 `ring_fill_` 代替 `ring_cap_` 做取模偏移，导致前 N-1 帧（缓冲区未填满时）指向错误位置。

### 修复后（CORRECT）

```cpp
int idx = (ring_head_ - 1 + ring_cap_) % ring_cap_;
```

无论缓冲区是否填满，最新帧总是 `ring_head_` 的前一个位置。

### 公式推导（ring_cap_ = 3，cap 为任意值时同理）

| 操作 | ring_head_ | ring_fill_ | 最新帧实际位置 | 旧公式结果 | 新公式结果 |
|------|-----------|-----------|--------------|-----------|-----------|
| 第 1 帧 push → 写入 [0] | 1 | 1 | [0] | (0+1)%3 = 1 ❌ | (0+3)%3 = 0 ✅ |
| 第 2 帧 push → 写入 [1] | 2 | 2 | [1] | (1+2)%3 = 0 ❌ | (1+3)%3 = 1 ✅ |
| 第 3 帧 push → 写入 [2] | 0 | 3 | [2] | else: (2)%3 = 2 ✅ | (2)%3 = 2 ✅ |
| 第 4 帧 push → 写入 [0] | 1 | 3 | [0] | else: (0)%3 = 0 ✅ | (0)%3 = 0 ✅ |
